# trusted-sender-mgmt Implementation Plan

**Goal:** Make the trusted-image-sender set consistent (add / contains / remove / list all route through ONE canonical normalization) and always auditable (a Settings "Remote images" section that renders unconditionally with an empty-state), without un-trusting any sender currently honored by the reader.

**Architecture:** A new PURE core `TrustedSenders` in `MMail/Mail/TrustedSenders.swift` (`Foundation` only ā€” NO AppModel / UserDefaults / Keychain / WebKit) owns `normalize(_:) -> String?` (the canonical lowercase ā†’ trim `.whitespacesAndNewlines` ā†’ strip `<>` ā†’ re-trim ā†’ nil-if-empty rule, identical to `AppModel.normalizeAddress`, `AppModel.swift:801-807`) plus `add`/`remove`/`contains`/`list` over an injected `Set<String>`. This is the SC-002/003/004/005 automatable core, mirroring the `ProxyConfigState` / `ReaderImageLoadState` house pattern. `AppModel.normalizeAddress` is rewritten to DELEGATE to `TrustedSenders.normalize` (so there is exactly one rule body), and the three impure trust ops `isImageTrusted` (`AppModel.swift:868-871`), `trustImages` (`873-878`), `untrustImages` (`880-883`) are refactored into thin wrappers that route through the core then mutate `trustedImageSenders` / persist to `UserDefaults`. Finally `SettingsView.swift:233` drops the `if !isEmpty` gate so the section always renders with an empty-state, copying the sibling "Blocked contacts" idiom (`SettingsView.swift:250-254`). Tests live in `MMailTests/` (fork-local, never goes upstream).

**Test Methodology:** e2e-first

**Test framework:** **Swift Testing** (`import Testing`, `@Suite struct`, `@Test func`, `#expect`) ā€” matches every existing file in `MMailTests/` (e.g. `ProxyConfigStateTests.swift`, `ReaderImageLoadStateTests.swift`, `ProxySecretStoreTests.swift`). Do NOT use XCTest.

**Pre-flight (handled by `/build`, not a task):** cut branch `feat/trusted-sender-mgmt` off `main` before editing any `*.swift`. Build runs on an Opus subagent; review is opposite-model (Sonnet).

**Command shorthand:**
- BUILD = `xcodebuild -project MMail.xcodeproj -scheme MMail -configuration Debug build CODE_SIGNING_ALLOWED=NO`
- TEST = `xcodebuild test -project MMail.xcodeproj -scheme MMail -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO`
- `xcodegen generate` runs ONLY in tasks adding a new `.swift` file (**T001, T002**); the regenerated `MMail.xcodeproj/project.pbxproj` (+ any scheme xcshareddata) is committed deliberately in that task's commit.
