# appearance-mode Implementation Plan

**Goal:** Replace MMail's binary `@Published var dark: Bool` with a THREE-way persisted **System / Light / Dark** appearance setting where **System** tracks the macOS system appearance live (re-rendering chrome, palette, AND open email bodies without relaunch), while every existing reader of `model.dark` keeps working unchanged because `dark` becomes a DERIVED value.

**Architecture:** A new pure, OS-free seam `enum AppearanceMode: String { system, light, dark }` (new file `MMail/State/AppearanceMode.swift`) owns three static, unit-tested functions — `resolvedDark(systemIsDark:)` (INV-2), `migrate(stored:legacyDark:)` (new-key-wins → legacy-bool → `.system`, INV-3), and `toggledExplicit(currentDark:)` (the `⌘⇧D` explicit-flip decision, never `.system`, INV-6). `AppModel` gains `@Published var appearanceMode` (persisted under the NEW additive key `mmail.appearanceMode`, written ONLY by `setAppearanceMode(_:)`) and recomputes the retained `@Published var dark: Bool` from `appearanceMode` + the live OS appearance, read from the system-wide signal `UserDefaults.standard.string(forKey: "AppleInterfaceStyle")` (NOT `NSApp.effectiveAppearance`, which reflects the app's forced scheme — INV-4). A SINGLE `DistributedNotificationCenter` observer on `"AppleInterfaceThemeChangedNotification"`, installed once for the AppModel lifetime, recomputes `dark` on the main actor when (and only when) `appearanceMode == .system` (INV-4/INV-5). `MMailApp` keeps driving `.preferredColorScheme`/`\.palette` off the derived `dark` unchanged. `SettingsView` swaps the on/off toggle for a segmented 3-way Picker; the `⌘K` palette gains a "System appearance" command. UserDefaults-only — no `MailCache`/`Email`/schema change (INV-8).

**Test Methodology:** e2e-first

### T5 — `AppModel`: install the single lifetime OS observer + the ⌘K "System appearance" command (INV-4, INV-5, INV-7)

  ```
  appearanceObserver = DistributedNotificationCenter.default().addObserver(
      forName: Notification.Name("AppleInterfaceThemeChangedNotification"),
      object: nil, queue: .main) { [weak self] _ in
  ```
