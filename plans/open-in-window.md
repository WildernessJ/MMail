# open-in-window Implementation Plan

**Goal:** Let a user pop a specific email into its own native, full-parity detached macOS window — bound to that one email's id, decoupled from the moving main-window selection, auto-closing when the email leaves its opener folder or is expunged.

**Architecture:** A second `WindowGroup(id:for: String.self)` scene keyed on `Email.id` (String) shares the single `AppModel` via `.environmentObject`. `AppModel` gains observable open/close *state* (a requested-id queue + a close-request set) but stays SwiftUI-window-free (INV-4); a view layer (`RootView`) holds `openWindow`/`dismissWindow` and drives the scene from that state. The detached window reuses the inline reader content (made accessible), captures its opener folder on first appear, and a pure close predicate plus pure lookup-by-id seam are unit-tested (SC-009).

**Test Methodology:** e2e-first

- **Automatable vs LIVE E2E tasks (build-subagent boundary):** every phase has an automatable `xcodebuild ... build` (or `build-for-testing`) gate — that gate is the build subagent's checkpoint and the only thing it can complete.
